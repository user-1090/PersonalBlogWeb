# 1.0 Ver  
  
---
  
## 一、框架搭建总体上大概历经了以下过程  
  

1. 把要求告诉gpt，让gpt生成建设指令;  

2. 把指令喂给copilot,生成初步页面;  

3. 把实际使用中遇到的问题扔给gpt和gemini，让他们生成解决方案或给copilot的修改指令;  
  

*实际操作中更倾向于直接生成解决方案，然后由我手动修改*  
  
*毕竟copilot太喜欢大刀阔斧……有改崩的风险*  
  


## 二、实际遇到的问题  
  

1. 博客无法正常显示；  
  
*原因：似乎是文件存储相对路径的问题*  
  
*解决方法：让gemini找出了需要修改的部分代码。其实在这里折腾了蛮久的……*  
  

2. markdown渲染有问题，空行无法正常渲染生成：  
  
*可以**fn+f12**先看看页面代码，发现虽然markdown文件里写了空行，但网页实际生成的时候代码中没有空行*  
  
*原因：Markdown 初始化代码：*  



    const md = window.markdownit({
    html: true,
    linkify: true,
    typographer: true,
    highlight: ...
    });


*缺了一个关键参数：*  

`breaks: true`  
  
  

## 三、学到的一些杂七杂八的东西  
  

### 博客写作方法：  
  

1. 在posts里创建新的.md文件，以markdown语法写作;  
  
2. 如有图片，将图片放入images文件夹内;  
  
3. 在文章中写入例如`![xxxx](images/first-blog/图片名)`的语句以插入图片;  
  
  

### 博客上传方法（含git）：  
  
  
    npm.cmd run built  
    
    git add .   
    
    git commit -m '提交名'  
    
    git push
